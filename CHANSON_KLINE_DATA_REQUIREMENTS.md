# 缠论多级别结构分析 - K线数据量需求研究

## 📊 缠论基本结构的数据需求

### 1. 最小数据需求（理论值）

#### 笔
**定义**: 连接相邻顶底分型的直线

**数据需求**:
- 最小K线数：**5根**
- 结构：底分型（3根）+ 顶分型（3根）- 重叠处理
- 实际需求：**7-10根**（包含关系处理后）

**计算**:
```
笔 = 顶分型（3根）+ 底分型（3根）= 6根
考虑包含关系处理：6根 × 1.5 = 9根
实际建议：10-15根
```

#### 线段
**定义**: 至少由3笔组成

**数据需求**:
- 最小K线数：**9根**（3笔 × 3根/笔）
- 实际需求：**15-20根**（包含关系处理）

**计算**:
```
线段 = 3笔 × 9根/笔 = 27根（理论最小）
考虑包含关系：27根 × 1.5 = 40根
实际建议：50-80根
```

#### 中枢
**定义**: 至少由3个线段重叠组成

**数据需求**:
- 最小K线数：**27根**（3线段 × 9根/线段）
- 实际需求：**40-50根**（包含关系处理）

**计算**:
```
中枢 = 3线段 × 27根/线段 = 81根（理论最小）
考虑包含关系：81根 × 1.5 = 120根
实际建议：150-200根
```

# 缠论多智能体分析系统

基于 LangGraph 和大语言模型的缠论全自动多智能体市场分析系统。

## 系统架构

### 已实现智能体

| 智能体 | 职责 | 状态 |
|--------|------|------|
| 数据采集智能体 | 获取Binance K线数据，检查数据质量 | ✅ 已实现 |
| 系统监控智能体 | 系统健康检查，数据质量监控，异常告警 | ✅ 已实现 |
| 模拟盘智能体 | 模拟交易执行，绩效统计，策略评估 | ✅ 已实现 |
| 首席决策智能体 | 综合研判，冲突仲裁，最终决策输出 | ✅ 已实现 |

### 工作流程

```
用户请求
    ↓
数据采集智能体 (获取K线数据)
    ↓
系统监控智能体 (检查系统健康)
    ↓
模拟盘智能体 (检查绩效状态)
    ↓
首席决策智能体 (生成交易决策)
    ↓
输出决策结果
```

## 功能特性

### 1. 数据采集
- 支持 Binance 交易所数据
- 多周期 K 线获取（1m, 5m, 15m, 1h, 4h, 1d）
- 数据完整性检查
- 数据质量评分

### 2. 系统监控
- CPU、内存、磁盘监控
- 智能体健康状态跟踪
- 数据新鲜度检查（5分钟内为新鲜）
- 异常告警机制

### 3. 模拟盘交易
- 初始资金 100,000 USDT
- 完整的开仓/平仓记录
- 实时绩效计算：
  - 胜率（Win Rate）
  - 总盈亏（Total P&L）
  - 盈亏比（Profit Factor）
  - 平均盈利/亏损
- 策略有效性评估

### 4. 首席决策
- 多周期联动分析
- 信号冲突仲裁
- 缠论核心理论应用：
  - 笔、线段、中枢识别
  - 背驰判断
  - 三类买卖点
  - 走势必完美
- 风险等级评估

## 决策优先级

1. **系统监控告警** - 最高优先级，任何告警暂停决策
2. **大级别结构** - 以大级别判断为准
3. **多周期共振** - 共振越多，置信度越高
4. **动力学确认** - 有背驰确认的信号优先
5. **模拟盘验证** - 胜率 < 45% 的信号降权
6. **风控否决** - 风控拥有一票否决权

## 缠论核心原则

系统严格遵循缠论理论体系：

1. **不臆测** - 结构未完成就说「未完成」，不强行定论
2. **不迎合** - 分析结论不因「希望市场涨/跌」而偏移
3. **不糊弄** - 有争议的结构标注「观察中」，不假装确定
4. **不甩锅** - 决策失误主动承认，不找借口
5. **不遗漏** - 关键风险必须上报，不得擅自忽略
6. **不造假** - 模拟盘结果如实记录，不美化
7. **不干扰** - 收到冷却信号后，立即停止生成信号

## 纪律要求

### 智能体自律
- 实事求是，数据缺失时明确说明
- 客观分析，不受情绪影响
- 风险控制，严格执行止损
- 先验后实，模拟盘验证后再考虑实盘

### 风控规则
- 单笔交易不得超过总资金的20%
- 连续亏损3次后暂停决策
- 胜率 < 45% 时降低仓位至50%
- 胜率 < 40% 时暂停决策，重新优化
